UNIT II LOGIC

Control Flow & Loops

Control flow allows your program to make decisions and repeat actions. Without it, code simply executes from top to bottom like reading a book.

2.1 Conditionals

Boolean Values

Expressions evaluate to either True or False.

They belong to type bool, they are NOT strings.
Operator Meaning
x == y Equal to
x != y Not equal to
x > y Greater than
x <= y Less or equal

Conditional Structures

IF
if expression:
  statement

Executes only if True.

IF-ELSE
if expr:
  do_this
else:
  do_that

Alternative execution path.

CHAINED
if expr1:
  ...
elif expr2:
  ...
else:
  ...

Checks multiple conditions.

2.2 Iteration (Loops)

While Loop

Repeats as long as condition is True. Use when iterations are unknown.

while expression:
  statement(s)

For Loop

Iterates over a sequence (List, String, etc.). Use for fixed number of times.

for var in sequence:
  statement(s)

What can you loop over?

Lists
Strings
Tuples
Dictionaries

* Nested Loops: A loop inside a loop (often used for multi-dimensional data).

2.3 Loop Control Statements

STOP

break

Terminates the loop immediately. Control exits the loop body entirely.

continue

Skips the rest of the current iteration. Jumps back to the start of the loop for the next round.

pass

A null statement. The interpreter executes it, but nothing happens. Used as a placeholder.

Logic on the Road

Think of Control Flow like Traffic Rules

Conditionals

Like Detours.
"IF the road is closed, take the detour; ELSE, drive straight."

Loops

Like Roundabouts or Race Tracks.
"Drive around until you run out of gas (condition) or finish 5 laps (count)."

Break

Like Parking the car.
You stop driving entirely and exit the road.

Continue

Like Swerving a pothole.
You avoid the bump but keep driving immediately after.
