5.1 Files

Text Files

Simple text, lines terminated by EOL (End of Line) \n.

Binary Files

Binary data readable only by computer. No specific line terminator.

File Modes

'r'
Read
Default mode
'w'
Write
Overwrites/Creates
'a'
Append
Adds to end
'x'
Create
Fails if exists
# Reading
f = open("file.txt", "r")
content = f.read()
f.close()

# Writing (Remember to close!)
f = open("file.txt", "w")
f.write("Hello")
f.close()

5.2 Command Line Arguments

Inputs given before the script starts.

sys

sys.argv

Simple list of arguments. sys.argv[0] is the script name.

Preferred
argparse

argparse module

Robust way to parse args. Handles help messages, types, and defaults.

getopt

getopt module

Similar to C style. Good for specific option parsing.

5.3 Errors & Exceptions

Common Exceptions

  • ZeroDivisionError: Division by zero.
  • ImportError: Module not found.
  • IndexError: Index out of range.
  • TypeError: Combining unrelated types.
  • KeyError: Key missing in dictionary.

Handling Pattern

Try

Code that might crash

Error?
Except

Handle the error here

Always
Finally

Cleanup (Close files)

5.4 Modules

Python files (.py) containing definitions. Breaks large programs into manageable pieces.

datetime
time
os
calendar
math

5.5 Packages

A directory containing modules. Must contain __init__.py to be treated as a package.

Directory Structure

Game/
__init__.py
Level/
__init__.py
start.py
load.py
Sound/
# Import examples

import Game.Level.start

from Game.Level import start

from Game.Level.start import difficulty

The Library Analogy

Organizing Code like a Library System

Module = Single Book

Contains specific info (Functions/Variables) on a topic.

Package = Shelf

A section (e.g., "History") grouping related books together.

__init__.py = Label

The Label on the shelf. Without it, Python thinks it's just a random pile, not an organized section.
